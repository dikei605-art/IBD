Diagnostic Scenarios

실제 진단 시나리오

원격 진단이 가능한 원리와 현장 엔지니어의 실제 워크플로우

CASE 01
체중값이 이상해요 — 환경 문제 vs 로드셀 문제 구분
체중 모듈
진단 원리
1
데이터 수집
부팅 시 무부하 상태의 내부저항·제로 ADC 자동 수집 — 별도 조작 없이 로그 누적
2
판정 로직
내부저항 이탈 → 로드셀 보드 불량 확정 ADC 이탈 → 로드셀 또는 로드셀 보드 의심 [외부충격 의심] Coef 이탈 → 로드셀 간 편차 발생 확인 [체중값 이상 의심] 편심 측정 → 특정 로드셀 불량 확정
3
조치
전 항목 정상 → 환경 요인 판단, 재부팅으로 자동 보정
엔지니어 워크플로우
1
CS 접수
"체중값이 이상해요"
2
원격으로 내부저항·ADC·Coef 이력 확인
→이상 있음: 로드셀 보드 불량 또는 로드셀 이상 확정
→정상: 환경 문제(장비 이동·외부 충격) 가능성
고객에게 껐다 켜기 안내 후 재측정
3
필요 시 편심 5위치 진단 실행
특정 로드셀 불량 여부 확정
4
판정 후 방문 여부 결정
5
진단 결과 CS 이력 저장
부품명·판정 근거·조치 내용 기록
이 케이스로 달라지는 것
기존: 방문 후 로드셀·보드를 순서대로 교체하며 원인 탐색
원격: 로그 확인만으로 방문 전 교체 부품 확정
CASE 02
인바디 측정이 안돼요 — 아날로그보드 문제 vs 전극보드 + 케이블 문제 구분
측정값 모듈
진단 원리
1
진단 방식 — Current O/X
인체지그 없이 장비 내부 전류 루프 형성 → 신호 흐름 확인. 전압 조절로 전류 드라이빙 정상 여부까지 측정
2
판정 로직 — 구간별 적용
ANO Current O/X → 아날로그 보드 적용 → 조건표 대조 → 정상/불량 판정 Current O/X → 전극보드·케이블 구간 적용 → 조건표 대조 → 정상/불량 판정
3
결론
두 구간 결과 조합 → 교체 필요 부품 1개 확정 후 방문
엔지니어 워크플로우
1
CS 접수
"측정 화면으로 아예 안 넘어가요"
2
원격으로 셀프 진단 실행
→아날로그 보드 이상: 보드만 지참 방문
→보드 정상 + 전극보드 체크 이상: 케이블 또는 전극보드만 지참 방문
→전체 정상: 장비 문제 아님 → 환경 확인
3
필요 자재 확정 후 방문
4
교체 완료 후 진단 재실행 · 이력 저장
정상 확인 후 CS 케이스 종결 처리
이 케이스로 달라지는 것
기존: 방문 후 여러 자재 교체하며 원인 탐색
원격: 방문 전 원인 확정 → 필요 자재 1개만 지참
CASE 03
갑자기 측정값이 이상해졌어요 — 노이즈 원인·방향 특정
노이즈 모듈
진단 원리
1
데이터 수집 · 트리거
대기 상태에서 사지 절대값 주기적 수집 → 임계값 초과할 경우 원격 진단시 알림 발생
2
판정 로직
경로: 팔 vs 다리 절대값 비교 → 공중 방사 vs 바닥 전도 구분 방향: 좌우 절대값 차이 → 노이즈 유입 방향(좌/우) 특정
3
조치
경로·방향 특정 결과 → 고객에게 구체적 위치 이동 안내 → 방문 없이 해결
⚠ 차분값만 보면 균일 노이즈를 정상으로 오판 가능
→ 절대값 개별 확인 필수
엔지니어 워크플로우
1
CS 접수
2
원격 진단 실행 → 방향 특정
공중/바닥 · 좌/우
3
고객에게 위치 이동 안내
위치 이동 / 필요시 노이즈 필터 제공
4
현장 방문 없이 해결
이 케이스로 달라지는 것
기존: 원인이 환경인지 장비인지 현장에서야 확인 가능
원격: 노이즈 경로·방향 특정 후 고객에게 즉시 구체적 위치 이동 안내
